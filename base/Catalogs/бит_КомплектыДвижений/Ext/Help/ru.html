Справочник «Комплекты движений»

 

1.      Назначение справочника

 

Справочник «Комплекты движений» является обобщающей сущностью, предназначенной для хранения состава шаблонов движений. Именно комплект движений, можно назначить в качестве обработчика проведения документа. Таким образом, комплект движений, с точки зрения первичных документов, является источником обработчика проведения. Отдельный шаблон движений являться источником не может.

Для использования одного или нескольких шаблонов движений вместе (для формирования движений одного вида документа), необходимо сформировать из них комплект движений, т.е. создать новый комплект движений и включить в него шаблоны движений, или добавить шаблоны движений в уже имеющийся комплект шаблонов.

Шаблон движений может описывать логику формирования движений только для одного регистра, комплект же может содержать в себе ссылки на шаблоны для разных регистров, таким образом, определяется более гибкая настройка обработчиков проведения. Например, можно разные сочетания шаблонов (комплекты) установить активными для разных организаций, либо менять активность комплектов с течением времени, без доработки самих шаблонов.

Комплект движений содержит в себе итоговый код обработчика и состав переменных, используемых в обработчике проведения. Данный код и состав переменных могут редактироваться пользователем вручную, если для комплекта отключено автоматическое изменение (интерпретация при изменении состава комплекта или при импорте комплекта).

Каждый комплект движения, перед использованием должен пройти процедуру утверждения (реквизит «Статус»). Неутвержденные комплекты движений не могут использоваться при проведении документов.

 

2.      Описание реквизитов

 

Назначение реквизитов и элементов формы справочника:

·         Номер - номер комплекта;

·        Вид документа - вид документа, для проведения которого предназначен данный комплект. Реквизит обязателен для заполнения. При заполненном реквизите, системой накладывается ограничение на перечень выбираемых шаблонов движений в состав комплекта (могут быть выбраны только шаблоны, для проведения именно этого вида документов);

·        Идентификатор - идентификатор комплекта, определяющий его уникальность при импорте/экспорте, в случае, если для одного вида документа существуют несколько комплектов движений;

·        Статус - статус комплекта (Проект/Утвержден);

·         Утвердил - ответственный, утвердивший данный комплект для использования;

·         Дата утверждения - дата утверждения комплекта (комплект не может быть использован для документов датой раньше, чем дата своего утверждения);

·      Проводки - список шаблонов движений, которые входят в состав данного комплекта движений (например шаблоны: БУ, НУ, МУ);

·         Отключить/Включить автоматическое изменение - отключает или включает возможность ручной корректировки обработчика движений и состава переменных, используемых в обработчике.
