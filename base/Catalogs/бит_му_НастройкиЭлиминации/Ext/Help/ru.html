Справочник «Настройки элиминации»


Консолидация – это такое представление отчетности группы из нескольких компаний, как будто это – одна компания.


Одна из важнейших задач в ходе процедуры консолидации – исключение влияния внутригрупповых операций на показатели отчетности (элиминация).


Обычно исключаются взаимные задолженности, выручка и затраты по внутригрупповым операциям, а также нереализованная прибыль от внутригрупповых операций в остатках активов, приобретенных по таким операциям.


Справочник «Настройки элиминации» предназначен для хранения шаблонов элиминационных корректировок, необходимых при формировании консолидированной отчетности. Настройки делятся на три типа: «Простой», «Выручка ВГО», «Нереализованная прибыль».


Рассмотрим каждый тип подробнее.


Шаблоны по удалению нереализованной прибыли анализируют стоимость поступивших, но еще не реализованных запасов. Разница в стоимости товаров, реализованных внутри группы (цена покупки - цена продажи), и образует нереализованную прибыль до тех пор, пока товар не реализован за пределы группы. Пользователю необходимо установить счета учета МПЗ, по которым будет производиться анализ нереализованной прибыли, а также счет учета затрат на реализацию у компании-продавца, который должен быть уменьшен («счет дополнительного движения»).


Пример настройки шаблона на удаление нереализованной прибыли по счету 2031 «Сырье и материалы», в условиях того, что себестоимость реализации у компании-продавца отражалась по счету 7013 «Себестоимость продаж»:


В шапке элемента:

·        Группа (элементы справочника можно объединять в группы);

·        Наименование (текстовое поле);

·        Регистр бухгалтерии (определяем регистр бухгалтерии, для которого будут действовать шаблоны элиминационных корректировок).


В элементе справочника представлены два табличных поля.


Первая табличная часть содержит следующие колонки:


·        Тип алгоритма элиминации (перечисление:Простой, Выручка ВГО, Нереализованная прибыль. Тип алгоритма определяет общий принцип работы шаблона элиминации);

·        Источники данных (выбирается источник данных, который позволяет сформировать отбор по внутригрупповым операциям; существует предопределенный перечень источников, но всегда данные перечень можно расширить и создать свой элемент).

·        Счет дополнительного движения (для удаления выручки и себестоимости по операциям внутри группы в источнике данных указывается счет выручки и НДС, на аналогичную сумму формируется проводка по счетам себестоимости, который указывается в графе счет дополнительного движения).


В табличном поле «Отбор источника» отображаются параметры отбора источника данных, указанного в табличной части:


·        Поле(в качестве поля могут выступать: счета, субконто);

·        Вид сравнения (перечисление: равно, не равно, в списке, в группе из списка, не в списке, не в группе из списка, в группе, не в группе);

·        Значение(тип данных значения определяется полем отбора).


! Замечания.


1.       В механизме исключения нереализованной прибыли в настоящий момент не обслуживается корректно ситуация, когда в качестве счета запасов выступает счет готовой продукции, либо приобретенные запасы являются продукцией собственного производства у продавца, ввиду того, что себестоимость продукции не известна на момент проведения документов реализации, а становится известна только после закрытия месяца.


2.       Источник данных «МСФО.НереализованнаяПрибыль», выбранный в настройке, представляет собой элемент справочника «Источники данных» - то есть, содержит ссылку на запрос (сам запрос уже имеется в базе, текст этого запроса можно посмотреть и при необходимости скорректировать) и параметры извлечения нужной информации из этого запроса.

Шаблоны по элиминации типа «Выручка ВГО» удаляют суммы выручки (за вычетом НДС) и себестоимости по операциям с собственными контрагентами, входящими в периметр.  В настройке пользователь указывается счета выручки, НДС  в отборе источника, в качестве дополнительного счета пользователь указывает счет себестоимости. По счетам выручки в шаблоне отбираются обороты с собственными контрагентами, входящими в периметр организации (важно, чтобы был корректно заполнен не только периметр консолидации, но и регистр сведений «Собственные контрагенты (БИТ)» - см. ниже описание данного регистра). Через регистратор движения определяется сумма НДС по данной хозяйственной операции.  Сумма НДС необходима для того, чтобы в элиминирующей проводке сумма выручки была уменьшена на сумму НДС исходящего.
