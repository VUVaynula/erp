Обработка «Формирование графиков платежей по финансовым договорам»

 

Для расчета графиков платежей по договорам кредита/займа и договорам по предоставлению кредитных линий  используется обработка «Формирование графиков платежей по финансовым договорам».

 

Обработка может вызываться из документа «Дополнительные условия по договору» по кнопке «График - Создать финансовый график» или использоваться самостоятельно в качестве «кредитного калькулятора».

 

«Кредитный калькулятор»

 

При использовании обработки в качестве «кредитного калькулятора» необходимо указать:

·         Способ расчета – выбирается из справочника «Способы расчетов графиков платежей по фин.договорам»;

·         Сумму договора (лимита) – обязательно при расчете графика платежей по кредитным линиям;

·         Направление ДДС  - получение кредита/займа или выдача займа;

·         Вид графика – БДДС (для формирования графика платежей) или БДР (для формирования графика начислений);


В табличной части «График выдачи траншей» - дату поступления кредитных средств и дату погашения, сумму транша, месяц первого платежа и количество периодов выплат (указывается в месяцах, кварталах, годах – в зависимости от периодичности амортизации основного долга);


В табличной части «Составляющие платежа» - значения параметров для составляющих платежа: процентные ставки, периодичность амортизации основного долга, процентов, комиссии и т.п.

 

В табличной части «График платежей» по кнопке «Сформировать график» будет сформирован график платежей или график начислений (в зависимости от вида графика). Красным цветом будут выделены даты платежей, попадающие на выходной день. По кнопке «Платежи выходных дней» можно перенести их либо на предыдущий либо на следующий рабочий день.

 

Примечание. Данные о рабочих и выходных днях хранятся в регистре сведений «Производственный календарь» (предварительно его необходимо заполнить).

 

Если даты платежей в графике превышают дату погашения транша, то будет выдано сообщение об этом и строки графика с такими датами будут выделены розовым цветом. В этом случае следует либо оставить график в таком же виде (если это приемлемо), либо скорректировать количество периодов выплат в табличной части «График выдачи траншей», либо изменить дату погашения транша.

 

В нижней части формы обработки выводятся итоги по графику – общая сумма поступлений и выплат по договору.

 

По нажатию кнопок в верхней части обработки можно управлять видимостью Параметров  и табличных частей «Итоги по графику» и «График выдачи траншей».

 

По кнопке «Печать» выводится печатная форма графика платежей:

 

Документ «Дополнительные условия по договору»

 

Для формирования графика платежей и графика начислений по договорам кредита/займа или договорам о предоставлении кредитной линии используется документ «Дополнительные условия по договору».

 

На закладке «График выдачи траншей» для договоров с типом «Кредиты полученные», «Займы полученные», «Займы выданные», а также для договоров с направлением «Финансовый» указывается график выдачи траншей.

При получении кредита – указывается дата получения денежных средств, дата погашения долга, сумма поступления, месяц первого платежа и количество периодов выплат.

Если банком открыта возобновляемая или невозобновляемая кредитная линия, то в табличной части «График выдачи траншей» указываются все транши, по которым нужно сформировать график погашения долга в данном документе. При этом можно вводить и отдельные документы «Дополнительные условия по договору» для разных траншей.

 

Примечание. Если по договору на предоставление возобновляемой или невозобновляемой кредитной линии предусмотрены пересекающиеся во времени транши, расчет графика платежей необходимо производить в одном документе «Дополнительные условия по договору». В противном случае неверно будет рассчитана комиссия за неиспользованный лимит.

 

На закладках «График платежей» и «График начислений» по кнопке «График – Создать финансовый график» открывается форма обработки «Формирование графиков платежей по финансовым договорам».

 

В форме обработки уже заполнены некоторые реквизиты:

·         Договор и/или Проект договора;

·         Сумма договора (лимита) – из договора или проекта договора;

·         Способ расчета – из договора или проекта договора;

·         Направление ДДС – на основании вида операции документа;

·         Вид графика – БДДС при создании графика платежей и БДР – при создании графика начислений;

·         График выдачи траншей – из документа;

·         Составляющие платежа и значения их параметров – из Договора контрагента или Проекта договора с закладки «Параметры финансовых договоров».

 

Для формирования графика достаточно нажать кнопку «Сформировать график» в табличной части «График платежей».  При необходимости можно перенести даты, приходящиеся на выходные дни, на предыдущий или следующий за выходным рабочий день.  Для этого используются кнопки «Перенести на следующий рабочий» и «Перенести на предыдущий рабочий» в подменю «Платежи выходных дней».

 

Если в обработке «Формирование графиков платежей по фин. договорам» изменены некоторые значения параметров составляющих платежа, то можно сохранить их в системе для выбранного договора (или проекта договора) по кнопке «Записать».

 

Для загрузки данных в документ «Дополнительные условия по договору» необходимо нажать кнопку «Загрузить» или «Добавить» в подменю «Перенести».  По кнопке «Загрузить» данные будут загружены в документ, при этом ранее загруженные или введенные данные будут удалены, по кнопке «Добавить» данные будут добавлены в документ, дополнительно к ранее загруженным или введенным данным.

 

Если ранее в документ «Дополнительные условия по договору» был загружен график платежей, и для нескольких строк графика уже были созданы документы планирования (заявки или планируемые поступления ДС) или документы прогнозирования (прогнозы платежей), то при загрузке данных из обработки «Формирование графиков платежей по фин. договорам» такие строки удалены не будут.

 

По кнопке «Печать» выводится печатная форма графика платежей.

 

В системе предусмотрена возможность корректировки графиков платежей и графиков начислений по финансовым договорам при изменении условий договора, например, при изменении процентной ставки или срока кредита.

Если с определенного момента произошло изменение какого-либо параметра договора, то график платежей (начислений) на оставшийся период может быть пересчитан с помощью обработки «Формирование графиков платежей по фин. договорам». В документе «Дополнительные условия по договору» необходимо нажать кнопку «График – Создать финансовый график», в форме обработки рассчитать график на оставшийся срок договора с учетом изменившихся значений параметров и загрузить (или добавить) в документ.
